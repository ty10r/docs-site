Advanced Usage

The topics under this document describe the underlying architecture and the advanced configuration settings for the Runtime Manager Agent.

For the vast majority of use cases, all you’ll need to do with the Runtime Manager Agent is install it and configure it according to what you can read in Installing and Configuring the Runtime Manager Agent

If you wish to use the Runtime Manager Agent for more than managing the communication between the Runtime Manager and Runtimes on-premises, then you may find the documents in this section helpful:

  • Runtime Manager Agent Architecture

  • Agent API

  • Event Tracking

  • Internal Handler Buffering

  • Extending the Runtime Manager Agent

None of these topics relate to the Mule Management Console. These topics are about the Runtime Manager Agent that communicates the Runtime Manager with local runtimes.
